题目描述

给定一个字符串 s，找到 s 中最长的回文子串。你可以假设 s 的最大长度为 1000。
例如

实现思路

方法一：暴力算法

枚举所有字串，花费为$O(n^2)$，判断是否为回文串，花费为$O(n)$。总的时间复杂度为$O(n^3)$。显然字符串最大长度为1000，该算法是无法通过的，但也不失为一种正确的算法。

方法二：动态规划

考虑一个回文字串$s’$，其为字符串$s$的下标为$[i,j]$的字串。若$s[i-1]=s[j+1]$，那么可以认为s的下标为$[i-1,j+1]$的字串也为回文字串。
定义$dp[i][j]$为字符串$s$的下标为$[i,j]$的字串是否为回文串，是则$dp[i][j] = true$，不是则$dp[i][j] = false$。
于是我们可以得到状态转移方程：
$$ dp[i][j]=\left{
\begin{array}{rcl}
true & {i = j} \
true & {s[i] == s[i+1]} \
dp[i + 1][j - 1] && s[i] == s[j] & other \
\end{array} \right. $$
首先，dp数组的边界条件为

  1. 所有元素本身构成回文，即 $i = j$ 时，$dp[i][j] = true$。
  2. 所有两个相邻元素相同构成回文，即$s[i] == s[i+1]$时，$dp[i][j] = true$。

  为了获得最长的回文串本身，需要记录回文串开始位置的索引，以及回文串的最大长度。在初始化时，设置最大长度为1，起始位置为0 (处理边界条件，如$s:”a”$；另外记得判空)。
  在进行边界处理时，若出现相邻元素为回文串的情况也需要记录起始位置以及最大长度为2(这是因为在动态规划填充dp数组时无法处理此种情况)。
  进行动态规划填充数组时，若出现比当前最大长度还长的回文字串，需要将起始位置和最大长度进行更新。另外，动态规划时，需要特别注意填充数组的顺序。状态转移方程中，计算$dp[i][j]$需要使用到$dp[i+1][j-1]$的值，因此需要从下向上遍历dp数组，才能利用到$dp[i+1][j-1]$(即当前元素左下角的元素)的值。

方法三：中心扩展

中心扩展时，可以将暴力算法中的枚举子串和判定回文同时进行。对于每一个原始字符串$s$来说，有$2n-1$个中心。为什么是$2n-1$个中心呢？这是因为不但每个字符本身可以当作中心，像$”bb”$这种字符串，两个$b$之间也可以当作中心。因此我们把每个字符串进行填充，填充的字符不能在原始字符串中出现过。例如$”abdbe”$可以填充为$”a#b#d#d#e”$，这样包括$”#“$的每一个字符就都能作为中心向两边扩展了。
